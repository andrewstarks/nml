nml, NanoMSG library for Lua

Version .2 alpha

NOTES ON REFACTOR

Right now, nml.new() is the method for creating a socket.

Instead, it should be:

nml[socket_protocol](s_options)

where s_options is an optional table of socket options to set.

An added feature is that we need to implement protocol level options and check that the correct options are set/get for the specific socket type.

Introduction: What is nml?

NML is a nanomsg binding for version 5.2 of the Lua programming language.

What is nanomsg?

Lifted from the nanomsg website...

nanomsg is a socket library that provides several common communication patterns. It aims to make the networking layer fast, scalable, and easy to use. Implemented in C, it works on a wide range of operating systems with no further dependencies.

nanomsg is lightweight, has a simple design, and is extreme portability... Does that sound like a language implementation that you know and love? :)

About this documentation

Much of this documentation was copied from nanomsg's files. It's modified for the changes that were made to accomodate the "Lua" way of doing things, such as using true or false instead of -1, etc.

Status of nml

nml is a work in progress and I am actively working on it. Things that are required for 1.0:

  • Tests
  • Support for raw sockets
  • Support for select in luasocket
  • Support for transport-level options, such as tcp_no_delay

Things required for me to be happy with the project:

  • Implement a pluggable message object architecture to aid the transfer of Lua types.
  • Provide tutorials for different use cases, particularly multi-threaded control.
  • Create some friendly libraries, such as independent select/epoll/iocp modules and a high performance timer library.
  • Provide a sane way in which to pass Lua code (functions / userdata) between threads. Look at Google's Protobuff as a possibility
